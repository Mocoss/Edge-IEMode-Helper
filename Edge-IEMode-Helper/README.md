# Edge IE 模式完整解决方案

本项目提供两种成熟的 Edge 浏览器 IE 模式启用方案，解决企业旧系统、海康威视等 ActiveX 控件网站在现代浏览器无法使用的问题。所有方案均基于微软官方接口，无第三方依赖，稳定可靠。

## 📋 两种方案对比

| 对比项 | 方案A：企业模式自动配置 | 方案B：原生右键切换 |
|-------|------------------------|---------------------|
| 操作方式 | 点击扩展图标一键完成 | 右键菜单手动切换 |
| 生效方式 | 访问站点自动切换 | 手动点击切换 |
| 后台进程 | 需要常驻本地程序 | 无任何后台进程 |
| 站点有效期 | 永久有效 | 单次/临时30天 |
| 适合场景 | 固定访问多个旧系统 | 偶尔使用旧系统 |
| 推荐指数 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

## 🚀 快速开始

### 推荐：方案 B（零成本最稳定）
1. 进入 `solution-b-native-right-click` 目录
2. 双击 `启用IE模式右键菜单.reg` 导入注册表
3. 重启 Edge 浏览器即可使用

### 方案 A（自动化程度高）
1. 编译本地宿主程序：参考 `solution-a-auto-sitelist/host-program/编译说明.md`
2. 配置原生消息主机：导入注册表，修改路径为你本地实际路径
3. 加载扩展：Edge 扩展管理页 → 加载解压缩的扩展 → 选择 `edge-extension` 目录
4. 点击扩展图标即可自动添加站点并以 IE 模式打开

## 🔍 验证生效方法
1. 地址栏输入 `edge://compat/enterprise` 回车
2. 可查看已加载的企业模式站点列表
3. 点击「强制更新」可立即刷新站点配置（无需重启浏览器）
4. IE 模式生效时，地址栏左侧会显示蓝色小 IE 图标

## ❌ 避坑指南
1. **`--ie-mode-force` 参数已废弃**：新版 Edge 已移除该启动参数，使用会提示「不受支持的命令行标志」，请勿再使用。
2. **站点列表无法 1 秒刷新**：官方最小刷新间隔为 30 分钟，设置更小值会被忽略。需要立即生效请手动到 `edge://compat/enterprise` 强制更新。
3. **进程复用导致参数失效**：Edge 默认单进程架构，带参数启动时若已有主进程，参数会被直接忽略。
4. **注册表路径必须正确**：企业模式策略位于 `Software\Policies\Microsoft\Edge`，写错路径完全不生效。

## 📝 补充说明
- 所有方案仅支持 Windows 系统，Edge 稳定版 100 以上版本
- IE 模式依赖系统自带的 IE11 内核，确保系统未禁用 IE 功能
- 内网 HTTPS 自签名证书站点，建议添加到受信任的站点，兼容性更好
